## 🔄 2. แผนผังการทำงาน (System Workflow)

### 🟢 2.1 ฝั่ง Backend (หลังบ้าน)

**`server.js`**
- ทำหน้าที่เป็น **Web Server** ให้บริการหน้าเว็บและ API
- **API Endpoints หลัก:**
  - `POST /api/add`: รับข้อมูลจองคิว -> ส่งต่อให้ `firebase.js` บันทึก
  - `GET /api/queue`: ดึงรายการคิวที่รออยู่
  - `GET /api/history`: ดึงประวัติ (รองรับการค้นหาและ Pagination)
  - `DELETE /api/queue/:id`: กด "เสร็จสิ้น" (ย้ายสถานะเป็น completed)
  - `POST /api/queue/:id/skip`: กด "ข้าม" (ย้ายสถานะเป็น skipped)
  - `GET /api/check/:phone`: เช็คสถานะคิวด้วยเบอร์โทร

**`firebase.js`**
- เป็นตัวกลางคุยกับ **Google Firestore** (Database)
- **ฟีเจอร์เด่น:**
  - **Transaction Safe**: การรันเลขคิว (Queue ID) ใช้ Transaction เพื่อป้องกันเลขซ้ำกัน 100%
  - **Auto Reset**: มีระบบเช็คอัตโนมัติ ถ้าลบประวัติจนหมดเกลี้ยง จะรีเซ็ตเลขคิวกลับเป็น 0 ให้เริ่มนับ 1 ใหม่
  - **Optimized Query**: การค้นหาเบอร์โทรใช้การ Query เจาะจงเพื่อความเร็วสูงสุด

### 🔵 2.2 ฝั่ง Frontend (หน้าเว็บ)

#### 1️⃣ หน้าลงทะเบียน (`index.html`)
- **หน้าที่**: ฟอร์มกรอกข้อมูลลูกค้า (ชื่อ, เบอร์, Order, ช่องทาง, สินค้า)
- **การทำงาน**:
  - ส่งข้อมูลผ่าน `POST /api/add`
  - มีรายชื่อแอดมินพร้อมอีโมจิ (🧸, 🍬, 🍭) ให้เลือก

#### 2️⃣ หน้า Dashboard (`dashboard.html`) - *ศูนย์บัญชาการ*
- **หน้าที่**: หน้าจอหลักสำหรับแอดมิน/แม่ค้า
- **Layout**: แบ่งซ้าย-ขวาเท่ากัน (50/50) เต็มจอพอดี (Fit Viewport)
  - **ซ้าย (คิวรอ)**: แสดงรายการคิวที่รออยู่ เรียงตามเวลา
  - **ขวา (ประวัติ)**: แสดงรายการที่เสร็จ/ข้ามแล้ว (เรียงต่อท้ายลงมา 1, 2, 3...)
- **การทำงาน Real-time**:
  - ใช้ **Firestore `onSnapshot`** (Listener) เชื่อมต่อตรงกับฐานข้อมูล
  - เมื่อมีการเปลี่ยนแปลง (เพิ่ม/ลบ/แก้ไข) หน้าจอจะอัปเดตเองทันทีโดยไม่ต้องรีเฟรช
- **ฟีเจอร์พิเศษ**:
  - **ค้นหาทันที**: พิมพ์ปุ๊บเจอปั๊บ (Client-side Search) ไม่ต้องรอกด Enter
  - **เลขคิวถาวร**: แสดงเลข ID จริงของคิว (เช่น #87) ไม่เปลี่ยนไปมา
  - **Scrollbar แยก**: เลื่อนดูรายการซ้าย-ขวาได้อย่างอิสระ

#### 3️⃣ หน้าเช็คสถานะ (`customer.html`)
- **หน้าที่**: ให้ลูกค้ากรอกเบอร์โทรเพื่อดูสถานะตัวเอง
- **การทำงาน**:
  - **แสดงเลขคิวตัวเอง**: โชว์เลข ID จริง (เช่น #87)
  - **สถานะปัจจุบัน**: บอกว่าตอนนี้แม่ค้ากำลังทำคิวที่เท่าไหร่ (เช่น #85)
  - **Countdown**: คำนวณให้เสร็จสรรพว่า "รออีก 2 คิว"
  - **แจ้งเตือน**: ถ้าถึงคิวแล้วจะขึ้นสีเขียวพร้อมข้อความ "🎉 ถึงคิวของคุณแล้ว!"

## 💡 3. เกร็ดความรู้ทางเทคนิค (Technical Highlights)

1. **Real-time Synchronization**:
   - ระบบใช้ `onSnapshot` ของ Firebase SDK ในหน้า Dashboard และ Customer ทำให้ข้อมูลตรงกันวินาทีต่อวินาที เหมือนแอปแชท

2. **Smart ID System**:
   - เลขคิวถูกจัดการที่ Server-side เท่านั้น เพื่อความถูกต้องแม่นยำ
   - ระบบ "Auto-Reset" เมื่อล้างประวัติ ช่วยให้เริ่มวันใหม่ได้ง่ายๆ แค่กดปุ่มเดียว

3. **Performance Optimization**:
   - **Client-side Search**: โหลดประวัติมาเก็บไว้ในเครื่องแม่ค้า (Cache) ทำให้การค้นหาชื่อลูกค้าเก่าๆ เร็วมากและไม่หนัก Server
   - **Lazy Loading**: ประวัติจะโหลดมาทีละส่วน ไม่โหลดทีเดียวทั้งหมดเพื่อประหยัดเน็ต

4. **User Experience (UX)**:
   - **Sticky Layout**: หน้า Dashboard ออกแบบให้เต็มจอพอดี ไม่ต้องเลื่อนหน้าจอหลัก เลื่อนแค่รายการข้างใน
   - **Visual Feedback**: มี Animation สีเขียว/แดง เมื่อทำรายการสำเร็จหรือพลาด
